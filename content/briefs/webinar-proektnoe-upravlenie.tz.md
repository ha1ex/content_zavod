## Summary

Лендинг регистрации на бесплатный вебинар «Проектное управление в Кайтене» (15 августа, онлайн, ведущий — Юра Юруков). Режим custom, тип `event_landing` — переиспользует компоненты эталонного `webinar-vnedrenie-kaiten` (форма регистрации в hero и финале, PainBubbles, TimelineRoadmap с номерами, SpeakerCard на градиенте, RegistrationCta). Цель страницы одна — заполнение формы регистрации.

Угол вебинара (согласован с командой 21.07.2026): **как выстроить проектное управление в Кайтене с нуля и совместить классику с гибкостью**. Классика — сроки, вехи, зависимости, диаграмма Ганта; гибкость — работа команды по канбан-доскам и спринтам поверх плана. Логика страницы: hero даёт «про что, для кого, когда» и сразу форму → блок «Знакомо?» проговаривает боль расхождения плана и работы → ключевой оффер разложен на два экрана (Ганта = классика, доски = гибкость) → программа показывает конкретику → промежуточная кнопка → блок ведущего → финальный CTA с формой. Форма в трёх точках: hero, кнопка-якорь после программы, финал.

## Аудитория

- **Основная:** Руководители проектов и проектные менеджеры, которым нужно вести проекты со сроками и вехами, не ломая привычную гибкую работу команды
- **Принимает решение:** Руководитель проекта / отдела — тот, кто отвечает за сроки и результат проекта
- **Роли:** Руководители проектов и проектные менеджеры, Руководители отделов и владельцы процессов, Тимлиды, ведущие проекты команды, Тёплые лиды на пробном периоде Кайтена, Те, кто присматривается к трекеру для проектов

**Боли (черновые, требуют подтверждения):**
- План проекта в одном файле, работа команды — в другом, и они давно разошлись
- Сроки плывут, а руководитель узнаёт об этом в последний момент
- У каждой команды свой инструмент — общей картины по проекту нет
- Диаграмма Ганта красивая, но через неделю неактуальна
- Статусы по проектам собираю вручную к каждому совещанию
- Жёсткий план тормозит команду, а без плана — хаос

## Структура лендинга

1. **Первый экран (hero) с формой регистрации** — За 5 секунд человек понимает — про что, для кого, когда, и видит форму. Три буллета «что заберёте» и строка ведущего. _(визуал: правую колонку занимает карточка формы регистрации.)_
2. **Узнавание боли «Знакомо?»** — Человек слышит свою ситуацию: план и работа разошлись, сроки плывут, статусы вручную. _(визуал: карточки-реплики облачками, проявляются по очереди.)_
3. **Ключевой оффер, часть 1 — классика** — Снять страх «в гибком трекере нет нормального плана». Собрать проект целиком: этапы, вехи, зависимости на диаграмме Ганта. _(визуал: диаграмма Ганта — `gantt-chart`.)_
4. **Ключевой оффер, часть 2 — гибкость** — Снять страх «жёсткий план убьёт привычную работу». Команда работает на своих досках, задачи сами встают в план. _(визуал: pm-доска — `pm-board`, зеркально.)_
5. **Программа вебинара** — Пять пунктов плюс хронометраж. _(визуал: вертикальный таймлайн с номерами.)_
6. **Промежуточная кнопка регистрации** — Закрыть тех, кого убедила программа. _(визуал: узкий градиентный баннер с кнопкой-якорем.)_
7. **Ведущий** — Доверие к спикеру. _(визуал: SpeakerCard на градиенте; фото нет → заглушка с инициалами.)_
8. **Финальный CTA + форма** — Повтор формы и мягкая строка под кнопкой. _(визуал: градиентный блок с формой.)_
