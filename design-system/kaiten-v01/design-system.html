Design System Kaiten V01 · site layout guide

Горизонтальный гайд по верстке лендингов

Сетки, токены, типографика, spacing, компоненты и схемы секций для методичной сборки лендингов Kaiten. Формат рассчитан на передачу дизайнеру, разработчику или агенту, который верстает страницу по дизайн-системе.

Главное правило

Страница собирается из контейнеров, сеток, секций и повторяемых компонентов. Никаких случайных декоративных решений без продуктового смысла.

Primary accent

#7D4CCF использовать для CTA, ссылок, активных состояний, фокусных иконок и важных подсветок.

Canvas

White / neutral sections, real UI-like previews, strong typography, clean B2B SaaS rhythm.

Desktop assembly

1. Создать frame 1920. 2. В каждой секции поставить centered container 1216. 3. Разложить контент по 12 колонкам. 4. Держать section padding 80/96. 5. Проверить, что CTA и product visual попадают в первый экран.

Mobile assembly

1. Frame 360. 2. Side padding 16. 3. Все группы переводить в vertical stack. 4. H1 text-3xl, body text-md. 5. Product visual не должен требовать зума, мелкий UI упрощается.

6. Rules For Built Landing Pages

Этот раздел описывает, как проверять уже сверстанный лендинг на соответствие Kaiten Design System V01 и референсам Landing-DS. Цель — чтобы страница была не просто похожа визуально, а методично собрана по сетке, токенам и компонентам.

Layout rules

  • Каждая секция имеет внешний full-width band: white, neutral-100, violet-soft или dark CTA.
  • Внутри секции всегда есть container: desktop 1216, tablet margin 24, mobile margin 16.
  • Header фиксирует ритм страницы: 80px desktop, 72px mobile.
  • Hero должен показывать продуктовый сигнал в первом viewport: UI-preview, board, dashboard, workflow.
  • Смена секций строится на чередовании белых и светло-серых блоков, без декоративных фоновых пятен.
  • Карточные сетки на desktop раскладываются 2x2, 3 columns или 4 columns; на mobile переходят в один vertical stack.

Token rules

  • Все цвета берутся из палитры DS: neutral scale, violet-100, violet-12 и semantic soft accents.
  • Primary CTA всегда violet-100 с белым текстом; secondary CTA — outline или white/neutral surface.
  • Текстовая иерархия: H1 48/60 desktop, H2 36/42, body 16/18, caption 12/14.
  • Spacing кратен шкале 4px; основные секционные отступы 80/96 desktop и 36/48 mobile.
  • Radius: buttons 8px, cards 16/24px, badges full, FAQ rows 12px.
  • Не вводить новые shadows, gradients и декоративные цвета без системной причины.

Component rules

  • Buttons не меняют размер между состояниями hover/focus/disabled.
  • FAQ имеет явные open/closed states: violet title/stroke в открытом состоянии, neutral в закрытом.
  • Product UI preview должен быть правдоподобным: колонки, карточки, комментарии, аналитика, документы.
  • Карточки содержат один смысловой фокус: icon/badge, title, body, optional link/CTA.
  • Footer не становится маркетинговой секцией; он завершает навигацию и поддержку доверия.

Responsive rules

  • На tablet сложные 12-column layouts переводятся в 6-column: 1/1 или 2-column блоки.
  • На mobile все сложные горизонтальные композиции становятся stack с gap 24/32.
  • Product visual на mobile упрощается, чтобы текст и UI не требовали зума.
  • Длинные CTA labels переносятся или сокращаются, но кнопка сохраняет системный размер.
  • Ни один текстовый блок не должен перекрывать следующий контент или выпадать из карточки.

Checklist before handoff

  • Desktop, tablet и mobile используют правильную сетку.
  • Контент каждой секции выровнен по контейнеру.
  • Нет случайных цветов вне токенов.
  • Violet используется как акцент, а не заливка всего интерфейса.
  • Типографика следует шкале text-xs ... text-6xl.
  • Spacing кратен шкале 4px.
  • Кнопки соответствуют размерам и состояниям.
  • Карточки имеют единый radius, padding и border.
  • Product UI preview объясняет продуктовый сценарий.
  • FAQ open/closed states различимы.
  • На mobile нет текста, который требует зума.
  • Секции можно повторить в коде без ручной подгонки.
