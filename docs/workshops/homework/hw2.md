# Домашка 2 — «Доведите свой лендинг до одобрения»

**Перед воркшопом 3. После того как на воркшопе 2 мы прошли intake → generate → ревью.**

Теперь самостоятельно доведи свой лендинг до статуса «одобрено». Здесь нормально
встретить замечания проверок — это часть процесса, а не провал.

---

## Шаг 1. Пройди весь круг

1. ТЗ из сырья: `/kaiten-intake` → ревью на `/intake/<slug>` → одобрить ТЗ.
2. Генерация: `/kaiten-generate`.
3. Превью: `pnpm dev` → `http://localhost:3000/landings/<slug>`.

## Шаг 2. Почини замечания проверок

Если агент «не пускает» — он всегда пишет, что именно не так. Самое частое:
- **Картинка-блок не из домена.** Поставь блок из своего домена (см. `domain-mock-matrix.md`).
- **Язык.** Англицизмы по-русски, без лозунгов и штампов.
- **Аудитория не покрыта.** Добавь сценарии/боли в бриф и перегенерируй.

Чини по кругу, пока проверки не станут зелёными. Подсказки по правкам —
`wiki/marketing/edit-recipes.md`. Если совсем застрял — `wiki/marketing/troubleshooting.md`.

## Шаг 3. Одобри

Открой `/approve/<slug>`, посмотри лендинг в браузере глазами клиента и поставь статус
«одобрено», если годится.

## Шаг 4. Выпиши вопросы

По ходу наверняка возникнут вопросы и блокеры — запиши их. На воркшопе 3 разберём
именно их, плюс научимся передавать готовое разработке.

---

## Что сдать

✅ **Лендинг в статусе approved** + короткий список вопросов/блокеров в общий чат.
