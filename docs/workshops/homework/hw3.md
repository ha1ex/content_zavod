# Выпускная работа — «Полный цикл соло»

**После воркшопа 3. Это финальная проверка: ты готов работать самостоятельно.**

Сделай новый лендинг **от начала до конца без подсказок** — от сырья до передачи
разработке. Бери другой продукт/возможность, не тот, что делал на воркшопах.

---

## Маршрут (весь путь сам)

1. **Сырьё.** Папка `inputs/<slug>/` с `request.md` и материалами.
2. **ТЗ.** `/kaiten-intake` → ревью `/intake/<slug>` → одобрить.
3. **Лендинг.** `/kaiten-generate` → починить все проверки.
4. **Ревью.** `pnpm dev` → `localhost:3000/landings/<slug>` → посмотреть глазами клиента.
5. **Одобрить.** `/approve/<slug>` → статус «одобрено».
6. **Передать.** `pnpm -w run harness handoff <slug> --require-approved`.
7. **Отправить в репозиторий.** Через Conductor / GitHub Desktop: commit → push → создать PR.
   (Сырой терминальный git не нужен — всё через кнопки.)

---

## Критерии «сдано»

- Лендинг открывается на `/landings/<slug>` и выглядит чисто.
- Все проверки зелёные.
- Статус на `/approve/<slug>` — «одобрено».
- `handoff` отработал, появился пакет для разработки.
- Изменения ушли в репозиторий (push + PR).

---

## Что сдать

🎓 **Ссылка на PR** + слаг твоего лендинга в общий чат.

Поздравляем — после этого ты делаешь лендинги самостоятельно. Дальше: помогаем друг
другу, копим уроки в `wiki/lessons.md`, а сложные доработки отдаём разработчикам.
