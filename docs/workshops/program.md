# Программа обучения команды: «Контент-завод Кайтен»

Гайд фасилитатора. Текстовая версия программы — источник истины для презентации
[`program.html`](./program.html). Аудитория: нетехническая маркетинг-команда (писатели,
редакторы, маркетологи, дизайнеры).

## Главная идея

Тяжёлую часть — код страницы — пишет агент. Человек отвечает за **качественный бриф,
честное ревью и решение «годится / правим»**. Поэтому обучение минимизирует трение
терминала и git, и максимизирует навык «общаться с агентом простым русским текстом».

- **Основной инструмент — Claude Code.** Скиллы `/kaiten-intake`, `/kaiten-generate`,
  `/kaiten-review` живут только в нём. Codex/GPT без этих скиллов — поэтому не как primary.
- **git — через GUI:** Conductor (этот воркспейс уже в нём) или GitHub Desktop. Сырой
  терминальный git команде не нужен.
- **Без API-ключей.** Хост-LLM — это ассистент в чате.

## Маршрут

```
ДЗ-0 → Воркшоп 1 → ДЗ-1 → Воркшоп 2 → ДЗ-2 → Воркшоп 3 → Выпускная работа
```

Перед каждым воркшопом — короткая домашка (можно закидывать заранее). Все домашки
лежат в [`homework/`](./homework/) с чек-листами и критерием «сдано».

---

## Домашка 0 — «Выравнивание базы» (перед воркшопом 1)

Self-paced. Цель — чтобы на первом воркшопе все были готовы. Подробно: [`homework/hw0.md`](./homework/hw0.md).

- Доступ к GitHub-репозиторию.
- Установлен Claude Code.
- Репо склонировано через Conductor / GitHub Desktop.
- `pnpm install` выполнен.
- `pnpm dev` запускается → открывается `http://localhost:3000`.
- Прочитана редполитика `wiki/brand/redpolitika.md`.

**Сдано =** скриншот работающего `localhost:3000`.

---

## Воркшоп 1 — «Карта местности + первый лендинг руками» (~2 ч)

Цель: снять страх и показать, как всё работает; каждый своими руками собирает первый
демо-лендинг.

**Блок A — понятия без страха (~40 мин)**
- Что такое репо / git / GitHub — на пальцах (аналогия: Google Docs с историей версий).
- Что такое Claude Code и зачем «говорить агенту по-русски».
- Большая картина завода: бриф → intake → лендинг → ревью → передача.
- Тур по веб-интерфейсу: `/landings`, `/intake`, `/approve`.

**Блок B — руками (~60 мин)**
- Запустить `pnpm dev`; посмотреть каталог блоков `pnpm storybook`.
- Запустить `/kaiten-generate` на готовом брифе `content/briefs/kaiten-factory.json`.
- Увидеть авто-выбор пайплайна; открыть превью `localhost:3000/landings/...`.
- Все генерят один и тот же демо-лендинг — для уверенности.

**Итог:** каждый своими руками собрал первый лендинг.
**Домашка после:** [`homework/hw1.md`](./homework/hw1.md).

---

## Воркшоп 2 — «Фабрика ТЗ + генерация своего лендинга» (~2.5 ч)

Цель: пройти ядро потока на **своём** материале.

**Блок A — фабрика ТЗ (intake)**
- `/kaiten-intake` на сырье из ДЗ-1.
- Ревью ТЗ на странице `/intake/<slug>`.
- Редполитика на практике: §10 англицизмы по-русски, §9 без лозунгов/штампов,
  §8 боли как управляемое улучшение.
- Одобрить ТЗ.

**Блок B — генерация + ревью**
- `/kaiten-generate` от своего брифа.
- Читать подсказки проверок, чинить по кругу (repair-loop).
- Превью `/landings/<slug>`, визуальное ревью.
- Страница одобрения `/approve/<slug>`.

**Гейты простыми словами (почему агент «не пускает»):**
- блок-картинка должна быть из домена продукта (нельзя ставить доску задач в CRM);
- язык — по редполитике;
- аудитория должна быть покрыта по сценариям (audience-score).

**Домашка после:** [`homework/hw2.md`](./homework/hw2.md).

---

## Воркшоп 3 — «Доработка, передача и самостоятельность» (~2 ч)

Цель: научиться отгружать готовое и владеть репозиторием.

**Блок A — итерации**
- Запросить правки у агента, перегенерить.
- Цикл управляемого улучшения: что менять в брифе/ТЗ, чтобы поднять качество.

**Блок B — передача разработке**
- approve → `pnpm -w run harness handoff <slug> --require-approved` (пакет для разработки).
- git-часть **через GUI** (Conductor / GitHub Desktop): commit + push + PR. Без сырого терминала.
- Как готовое доходит до разработчиков.

**Блок C — владение**
- Самим править брифы и контент.
- Добавлять уроки в `wiki/lessons.md`.
- Подтягивать свежее (pull/sync), базовая гигиена веток.
- Куда обращаться за помощью; будущий трек статей.

**Выпускная работа:** [`homework/hw3.md`](./homework/hw3.md) — полный цикл соло.

## Ежедневный поток (шпаргалка)

| # | Шаг | Команда / экран |
|---|-----|-----------------|
| 1 | ТЗ из сырья | `/kaiten-intake` → ревью на `/intake/<slug>` → одобрить |
| 2 | Сгенерить лендинг | `/kaiten-generate` (агент сам выберет, как собирать) |
| 3 | Посмотреть | `pnpm dev` → `localhost:3000/landings/<slug>` |
| 4 | Одобрить | страница `/approve/<slug>` |
| 5 | Передать | `pnpm -w run harness handoff <slug> --require-approved` |

Под капотом шага 2: `pnpm -w run harness agent build landing --slug <slug> --brief content/briefs/<slug>.json`.
