# ComparisonTable

Таблица сравнения двух продуктов (Кайтен vs конкурент) с раскрывающимися
разделами и отметками наличия функций. Один макет, три брейкпоинта:
Desktop / Tablet / Mobile.

> **Дефолт пайплайна (правило `comparison-uses-mock`).** Для любого сравнения
> «Кайтен vs конкурент» в обоих флоу (по брифу и custom) секция `ComparisonTable`
> используется в **grouped-режиме** и рендерит эталонный `ComparisonTableMock`
> (`packages/ui/src/landing/mocks/ComparisonTableMock.tsx`). Props:
> `sections: [{ title, rows: [{ label, a, b }] }]` (a=Кайтен, b=конкурент),
> `competitor`, `footnote`. Плоский режим (`columns`+`values`) — legacy-fallback.
> Правило целиком — `packages/harness/src/prompts/content-factory-methodology.md`.

## Структура

- **title** — «Сравнение функций»
- **section** (таблица из колонок):
  - **колонка-метки** — заголовки разделов + названия строк
  - **колонка «Кайтен»** — выделенная: лиловый фон, логотип, галочки
  - **колонка конкурента** — серый фон, название продукта, галочки / прочерки
- **footnote** — источник и дата актуальности

Это не `<table>`, а набор flex-колонок с равным числом ячеек, выровненных по
высоте строк.

## States

- **Раздел раскрыт.** Стрелка вверх (chevron-up), заголовок брендовым лиловым
  `#7D4CCF`, строки видимы.
- **Раздел свёрнут.** Стрелка вниз (chevron-down), заголовок neutral `#2D2D2D`,
  строки скрыты.
- **Ячейка — функция есть.** Зелёная галочка в белом круге.
- **Ячейка — функции нет.** Серый прочерк «–» в белом круге.
- **Колонка «Кайтен».** Всегда акцентирована: фон `#EFE9F9`, скруглённый верх,
  логотип в шапке вместо текстового названия.

## Tokens

| Токен | Значение | Применение |
|-------|----------|------------|
| `text/title-color` | `#2D2D2D` | заголовки, текст строк |
| `text/brand-color` | `#7D4CCF` | заголовок раскрытого раздела |
| `text/tertiary-text-color` | `#9E9E9E` | сноска, разделители разделов |

Типографика — Roboto, letter-spacing `-0.2px`. Шкала `4xl 36/40` ·
`2xl 24/32` · `lg 18/28` · `md 16/24` · `sm 14/20` · `xs 12/16`.

## Usage rules

- Ровно два продукта; первый — всегда Кайтен (выделенная колонка).
- 1–5 разделов, в каждом 1–N строк функций. По умолчанию раскрыт первый раздел.
- На Mobile название конкурента сокращай (напр. «Юздеск») — колонка 78 px.
- Сноска обязательна: источник + дата актуальности, третичным текстом.

## Anti-patterns

- Не выделять колонку Кайтена цветом / логотипом (теряется акцент).
- Не оставлять пустую ячейку — всегда галочка или прочерк.
- Не сравнивать больше двух продуктов в этом компоненте — используй широкую
  pricing/feature-matrix раскладку.
