# Manufacturing landing — reference

> Эталон визуального и текстового тона для лендингов Кайтена под
> производственные компании. Источник — клиентский бриф «Кайтен для
> производств» с реальными скриншотами Кайтена в продакшен-аккаунтах
> производственных клиентов (Неос Ингредиентс, ГК Рэмэкс,
> Авиационный Консалтинг-ТЕХНО, ПФ-ФОРУМ).

## Когда выбирать этот reference

- Продукт: Кайтен (или похожая PM-платформа), позиционируется под
  производственные компании — мелкосерийное и контрактное производство,
  машиностроение, металлообработка, приборостроение, оборудование на заказ.
- Аудитория: гендиректор / директор производства / руководитель проектного
  офиса завода / главный технолог / технический директор.
- Боль: производство идёт по чатам и Excel, статусы заказов собираются
  обзвонами, чертежи лежат в почте и сетевых папках, отчёты собираются
  руками перед планёркой.
- Желание: один маршрут для каждого заказа (заявка → закупка → производство
  → ОТК → отгрузка), спецификации и чертежи внутри карточки задачи,
  загрузка цеха на недели вперёд, отчёты без ручной сборки.

## Доменно-специфическая лексика

Используй в копи. Прямой признак, что лендинг написан про производство:

- **Заказ**, не «задача» / не «тикет» / не «сделка».
- **Маршрут заказа:** заявка → закупка → производство → ОТК → отгрузка
  (можно сокращать до «закупка → цех → ОТК → отгрузка»).
- **Участок** / **цех** / **отдел** (Цех №3, Цех сборки, ОТК), не «команда».
- **Спецификация** / **чертёж** / **ТЗ** / **техкарта** — документы внутри
  карточки заказа.
- **Снабжение** / **технологи** / **мастера** / **кладовщики** — роли.
- **ОТК** (отдел технического контроля) — отдельная стадия, не «ревью».
- **Просрочка** / **узкое место** / **загрузка участка** — операционные
  метрики, не «velocity» / не «throughput».
- **Партия N шт** — единица заказа (партия 12 шт, серия 200 шт).
- **Планёрка** — формат, который мы заменяем. Раньше — обзвоны и Excel,
  теперь — взгляд на доску.

## Mock-набор домена

Все 5 mock'ов используют **реальные скриншоты Кайтена** для производственных
аккаунтов, оформленные window-chrome'ом для консистентности с остальной
библиотекой.

### `production-board` — Hero / MediaCopy / Tab / Scenario

Главный mock домена. Канбан-доска «Проект Альфа» с производственными
стадиями Очередь → В работе → Согласование ОТК → Правки ОТК, переработка →
Готово. Карточки заказов содержат каркасы, спутники, пластины с номерами
44.0056 / 24.0005 / 50, исполнителями (Кодынгин М., ИП Освентьев),
участками. Используй в Hero (`visualPosition: 'below'`) как «лицо
лендинга».

**Когда выбирать:** главная иллюстрация рабочего процесса производства.

### `order-flow` — MediaCopy / Tab / Scenario

Двойная доска «Заказы на поставку» + «Проект Омега» — внешние заявки от
клиентов и внутренний производственный цикл с одинаковой воронкой стадий
бок о бок. Подчёркивает, что внешние и внутренние процессы живут в одной
системе с единым языком статусов.

**Когда выбирать:** секция про связь снабжения и производства, или про
прозрачность сквозного потока от заявки клиента до отгрузки.

### `production-gantt` — Hero / MediaCopy / Tab

Диаграмма Ганта с фазами Инициация → Планирование → Реализация. Цветные
полосы показывают загрузку участков на февраль, видны пересечения
параллельных задач и зависимости. Иллюстрирует ресурсное планирование на
недели вперёд.

**Когда выбирать:** секция про планирование загрузки, ресурсное
планирование, видимость пересечений этапов.

### `production-task-card` — MediaCopy / Scenario

Карточка «Корпус насосной станции, партия 12 шт» для Проекта Альфа.
Содержит: цех №3, номер заказа A-2847, чек-лист 3 из 6 (спецификация,
фото материалов), исполнителей, срок 31 мая. Иллюстрирует «всё внутри
карточки» — без папок и почты.

**Когда выбирать:** секция про хранение документации, чек-листы по этапам,
карточку задачи как единое рабочее место.

### `production-departments` — MediaCopy / Tab

Mockup ноутбука с производственной доской и плавающими лейблами 6
департаментов: Бухгалтерия и финансы / Оценка рисков и планирование /
Логистика и закупки / Проектирование / Коммерческий отдел / Производство.
Иллюстрирует «вся компания в одной системе» — каждый видит свой контур
и общую картину.

**Когда выбирать:** секция про объединение департаментов, кросс-функциональную
видимость, замена ручной коммуникации между отделами.

## Layout

Используй [`single-module-deep-dive`](../layouts/single-module-deep-dive.md).
Производственный лендинг — это отраслевой deep-dive: одна аудитория, один
большой сценарий, конкретные доменные mock'и.

Hero `visualPosition: 'below'` обязателен — крупная производственная доска
сразу под заголовком отвечает на вопрос «что я увижу внутри?».

## Кейсы (фрагменты для SocialProof)

Реальные клиенты Кайтена в производственном сегменте:

- **Неос Ингредиентс** (Кирилл Флегонов, гендиректор) — «упростил контроль
  техпоручений, точно вижу сколько задач у каждого отдела, экономит время
  совещаний».
- **ГК Рэмэкс** (Борис Серов, директор) — «не нужно беспокоить сотрудников
  для уточнения статуса или сроков задач — всё отображено на досках».
- **Авиационный Консалтинг-ТЕХНО** (Василий Пыжьянов, технический директор)
  — «освобождает минимум два рабочих дня каждый месяц, больше не нужно
  вручную объединять отчёты».
- **ПФ-ФОРУМ** (Марина Вербицкая, интегратор) — «за 1,5 года ушли от
  записей в блокнотах к прозрачным канбан-доскам».

## Anti-patterns

- ❌ Reuse `pm-board` или `support-board` под производство. Лексика
  карточек принципиально другая (story points vs участок №3, sprint vs
  партия 12 шт). Используй `production-board` или `order-flow`.
- ❌ Слова из чужих доменов: «спринт», «эпик», «story points»,
  «velocity», «MR», «тикет», «обращение клиента» (это поддержка),
  «лид», «сделка», «воронка продаж» (это CRM). Производство говорит
  «заказ», «партия», «маршрут», «участок».
- ❌ Hero без `production-board` или `production-gantt`. Эти два mock'а —
  главные визуальные сигналы домена.
- ❌ Кейсы из чужих сегментов (Газпромбанк, Тинькофф) — не убеждают
  директора производства. Используй кейсы из таблицы выше или
  родственные (заводы, цеха, технические компании).
